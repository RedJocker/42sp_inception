*** Requirements
- [X] Each Docker image must have the same name as its corresponding service
- [X] Each service has to run in a dedicated container
- [X] You also have to write your own Dockerfiles, one per service.
- [X] The Dockerfiles must be called in your docker-compose.yml by your Makefile.
- [X] A container that contains NGINX with TLSv1.2 or TLSv1.3 only
- [X] A container that contains WordPress + php-fpm only, without nginx
  - [X] wordpress must be installed and configured
  - [X] A volume that contains your wordpress files
- [X] A Docker container that contains MariaDB only, without nginx.
  - [X] A volume that contains your database data
- [X] Your volumes will be available in the
      /home/login/data folder of the host machine using Docker
- [X] A docker-network that establishes the connection between your containers
- [X] Your containers have to restart in case of a crash.
- [X] configure your domain name so it points to your local IP address.
      This domain name must be login.42.fr
- [X] Wordpress database must have two users with one administrator
- [X] Nginx must be the only entrypoint to services via 443 using TLSv1.[23]
- Forbiden
  - [X] using latest tag on containers
  - [X] exponsing sensitive data(passwords, credenntials, keys)
  - [X] not using .env
